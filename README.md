## 產生器寫法
###### 方針
優先順序(上面優先) 
1. 方便使用 - 用起來舒服
2. 程式碼間少關聯 - 維護起來方便，不會互相卡
3. demo輸出樣式(文字即可) - 快速知道產出是什麼 
4. code易閱讀 - 好改
5. 減少冗於動作 - 不要浪費效能
6. 輸出整齊 - 品質
